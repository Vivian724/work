说明

Share Treat 商品在前台与其他积分商品展示一致，无新增字段。商品图片、商品简介、购买价格（积分）、积分折扣、库存、限时抢购标签等均由后台填写 / 带出（详见后台规格）。

字段（沿用）

字段来源说明
商品图片后台后台上传或三方带出
商品名称后台 / 三方如「实物」「VP道具卡」
商品简介后台详情弹窗正文
购买价格（积分）后台会员需支付的积分数
积分折扣 / 限时后台如「-5%」「划线原价」
库存后台 / 三方可显示剩余库存

新增：如何兑换 / 兑换条款入口

商品详情弹窗内新增一行蓝字「如何兑换 / 兑换条款 ›」，点击弹出「兑换说明」二级弹窗（见下方弹窗规格）。该内容为会员购买前必看，含使用方式与不可退等限制。

项目规格
位置商品详情弹窗内，价格下方、「购买」按钮上方
样式蓝色文字 + 右箭头「›」，可点击
点击行为弹出「兑换说明」二级弹窗（覆盖在商品详情之上）

「兑换说明」弹窗规格

字段来源说明
弹窗标题固定「兑换说明」或「如何兑换 / 条款」
正文内容三方 Brand API redeemInfoUrl含 ABOUT / HOW TO REDEEM / TERMS OF USE 等段落
关闭固定右上角「×」或点遮罩返回商品详情
条款内含「购买后不可取消 / 不可退款」「一次性使用」「仅限指定门市 / 不适用外送」等限制，是客诉高发点，必须购买前可见。建议「不可退」一句在弹窗内红字突出。
内容为英文 / 当地用语时，建议后台可对该段做本地化覆盖；无覆盖则原样显示三方内容。是否本地化由后台决定（与后台规格对齐）。

功能逻辑

  • 点「购买」进入确认订单（页面 2）。
  • 库存为 0 或商品下架时，「购买」按钮置灰不可点。
  • 「兑换说明」弹窗内容拉取失败时显示「加载失败，请稍后重试」+ 重试，不阻断浏览商品。

新增：兑换条款同意勾选

项目规格
文案「我已阅读并同意《兑换条款》，购买后不可退款」
《兑换条款》蓝字可点，弹出与页面 1 相同的「兑换说明」弹窗
默认状态不勾选（不可预勾，须会员主动勾选）
付款按钮未勾选时置灰不可点；勾选后恢复可点
放在「付款」上方而非页面 1，是为让同意动作紧贴不可逆的付款行为；条款含「不可退」，事后争议时更有依据。

可选加强（防盲勾）

  • 要求会员先点开过「兑换说明」弹窗，勾选框才可勾。适合「不可退」类商品，代价是多一步。是否启用由产品决定，建议先上基本版（仅默认不勾 + 付款置灰）。

功能逻辑

  • 未勾选同意：付款置灰；点击置灰按钮可轻提示「请先阅读并同意兑换条款」。
  • 「付款」前再次校验库存与积分余额；不足时提示并阻止下单。
  • 付款＝扣减会员积分并向后台 / 三方发起兑换（派发）。成功后进入购买成功（页面 3）。
  • 建议服务端记录本次「同意条款」的时间戳与条款版本，便于日后查证。
扣分与三方派发的时序、失败回滚（退积分）以后台 / 服务端为准；前台只负责呈现结果状态。

订单详情（既有页）：步骤条＋订单信息＋新增兑换信息区块

改动说明

订单详情为既有页面，本次不新增页面，仅在原有「步骤条 + 订单信息」之下新增「兑换信息」区块。该区块展示第三方（BIZ TREATS）派发回传的数据，是会员实际「拿到货」的地方。

关键区分：「订单编号」≠「兑换码」。「订单编号」是识别这笔交易、用于查单 / 客服 / 对账；会员拿去实体店核销的是「兑换码 / 兑换链接」，不是订单编号。两者在页面上分区块呈现，避免会员拿错。

① 订单信息区块（查单用 · 既有）

字段来源对应 API说明
订单编号（自家）自家系统= 传给三方的 referenceNo原「兑换单号」改称「订单编号」；接三方前后都由自家生成，建议直接作为 referenceNo 传给三方，保证 1:1 对应
商品名称 / 价格自家 / 后台—沿用
三方单号（可选）三方bizReferenceNo默认不在会员端显示，放后台对账（后台已有「查看三方信息」）；仅当客服流程需会员报三方单号时，才以折叠 / 次要信息加入
单号命名约定：自家＝「订单编号」（PR… ，referenceNo）；三方＝「三方单号 / 供应商单号」（260… ，bizReferenceNo，与后台用词一致）。避免两者都叫「单号」造成混淆。

② 兑换信息区块（换货用 · 新增）

以下字段全部来自三方派发回传，是会员去实体店核销的依据。

字段类型对应 API说明
兑换码 / 密码文本 + 复制pin码型：会员出示此码给店员核销；Universal 型作为兑换链接的 Passcode
兑换链接链接 + 复制 / 打开bizRedeemUrl连结型：会员打开此链接兑换，导向供应商兑换页
有效期至文本pinExpiredDate派发后的确切到期日，格式化展示，过期需提示

按商品类型（pinType）显示规则

类型pinType显示内容
PIN 码型pin仅显示兑换码（+ 有效期）
连结型（url）仅显示兑换链接（+ 有效期），可「打开链接」
连结＋密码型urlWithPin兑换链接 + 兑换密码（pin 作 Passcode），链接页需输入密码
三方按商品决定回传 pin / bizRedeemUrl 二者其一或并存；前台按实际回传字段渲染，不要写死。无值字段不展示。

按状态显示规则

状态步骤条兑换信息区块
待审核待审核（高亮）→ 已完成（灰）显示「审核中，发放后可查看兑换信息」，不显示码 / 链接
已完成两步均 ✓按 pinType 显示兑换码 / 链接 / 密码 + 有效期
失败失败态（红）显示失败原因摘要 + 「联系客服」；积分已退则提示「积分已退回」

功能逻辑

  • 复制：兑换码 / 链接 / 三方订单号均提供一键复制，复制后 Toast「已复制」。
  • 打开链接：连结型 / Universal 型提供「打开兑换链接」；Universal 型同屏提示「兑换密码：{pin}」。
  • 有效期：到期前可正常显示；已过期时码 / 链接置灰并提示「已过期，请联系客服」。
  • 手动审核：审核通过后才会有兑换数据；此前区块为「审核中」占位，审核通过后刷新自动展示（呼应列表「数据同步存在延迟」）。
  • 「联系客服」入口常驻。

异常处理

  • 派发失败（如三方余额不足 API_WALLET_INF0001、无库存 API_LOCATION_RED0001）：状态置「失败」，提示失败 + 退积分说明（以服务端为准）。
  • 兑换信息拉取失败：区块显示「加载失败，请稍后重试」+ 重试按钮；不影响订单其他信息展示。
  • 无任何兑换字段返回（异常）：区块显示「暂未获取到兑换信息，请联系客服」。
