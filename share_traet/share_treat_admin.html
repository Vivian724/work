积分商城　后台需求规格（Share Treat）

版本 v1更新日 2026-06-22读者：QA / RD

变更摘要

QA 验收范围：本次后台改动仅针对「Share Treat」类型商品，验收请聚焦 Share Treat 的新增商品 / 三级联动 / 库存取小值 / 审核（通过·拒绝退分）/ 查看三方信息（脱敏）等流程；其他商品类型不在本次范围、行为维持不变。

页面 1新增商品 路径：后台 > 积分商城 > 商品管理 > 新增商品
红框＝本次新增区块

改动说明

新增商品类别 Share Treat 及商品类型 Share Treat。选中商品类型 Share Treat 后，在下方显示一组「三级联动 + 商品详情」字段，供管理员选定要上架兑换的商品。其余字段（商品图片、购买价格、积分折扣比例、排序、单个会员限购数、库存、是否限时抢购、审核模式等）维持既有逻辑不变。

改动点

  1. 新增商品类别 Share Treat 及商品类型 Share Treat。
  2. 选中 Share Treat 后，新增字段（图中红框）：选择类别、选择厂商、选择商品、商品详情、商品简介。
  3. 选中其他商品类型时，上述新增字段不显示。

字段规格

字段类型必填说明
选择类别 Category下拉是Share Treat 商品类别，数据来自第三方 API
选择厂商 Brand下拉是依所选类别联动带出
选择商品 Goods下拉是依所选厂商联动带出
商品详情 Goods Detail显示（只读）—选定商品后自动带出；仅供运营参考，不可编辑
库存数字是管理员手动输入；保存时系统自动与第三方 API 回传库存对比，取两者较小值作为实际上架库存上限。若第三方 API 回传库存为 0 或无回传值，「库存」栏位自动置灰、不可填写
商品简介文本是沿用既有字段

功能逻辑

  • 联动顺序：类别 → 厂商 → 商品 → 详情。
  • 上层变更时，下层选项需清空并重新加载。
  • 类别 / 厂商 / 商品三级数据均来自第三方提供的 API（接口文件）。
  • 库存取最小值：保存时系统自动比较管理员输入数量与第三方 API 回传库存，取二者中的较小值作为最终上架库存，以防止超卖。
  • 若第三方 API 回传库存为 0 或无回传值，「库存」输入栏自动置灰、不可填写。

异常处理（第三方 API 连接失败）

选择 Share Treat 后，若三级下拉的数据来源（第三方 API）连接失败或无响应，需弹出错误提示，并将受影响的下拉置为不可选、「确定」按钮禁止提交。

  • 错误提示（顶部 Toast）：「资料载入失败，请检查网络或稍后重试」
  • 连接失败期间「确定」按钮禁止提交，避免存入空商品。

库存逻辑情境一览

情境管理员填写三方回传最终上架库存说明

B · 管理员填写 ≤ 三方库存 200 500 200 取管理员值（较小值）
C · 三方回传库存为 0 任意 0 0 「库存」栏位自动置灰，不可填写；最终库存为 0
D · 三方无库存数据 任意 null / 无 — 「库存」栏位自动置灰，不可填写；无法新增库存数据

改动说明

针对 Share Treat 类型商品，在类型筛选下拉新增选项，并在「操作」栏提供查看第三方（供应商）兑换信息的入口。弹窗样式参考既有「充值信息」弹窗。

改动点

  1. 类型筛选下拉（最右侧）：新增选项 Share Treat，可按此类型筛选。
  2. 操作栏：Share Treat 类型订单显示「查看三方信息」（对应原话费类的「查看充值信息」）。
  3. 审核控制：
    当订单为手动审核时，操作栏先显示「审核」按钮；点击并完成审核后，操作栏才显示「查看三方信息」。若为自动审核，操作栏直接显示「查看三方信息」。
  4. 点击「查看三方信息」：弹出「三方信息」窗口，显示三方订单号、兑换码 / 兑换链接。

审核动作（手动审核：通过 / 拒绝）

手动审核的订单，操作栏先显示「审核」按钮。点击后进入审核处理，管理员可选择通过或拒绝，并可填写「备注」（非必填，沿用既有逻辑）。审核结果与后续行为如下：

审核结果订单状态积分三方派发操作栏前台对应
通过审核通过维持已扣减触发三方派发兑换数据改显「查看三方信息」已完成
拒绝审核未通过全额退还本次扣减积分不派发、无兑换数据不显示「查看三方信息」已拒绝
拒绝＝全额退积分。审核拒绝后，系统自动将本次扣减的积分全额退还至会员账户（与其他商品类型既有逻辑一致）。前台该订单状态转为「已拒绝」，且不显示任何兑换码 / 链接。

订单状态与操作人

订单状态含义操作人显示
待审核手动审核、尚未处理—（待处理）
审核通过自动审核通过，或管理员审核通过自动＝SYSTEM；人工＝管理员账号（如 vivianadmin）
审核未通过管理员审核拒绝执行拒绝的管理员账号
「操作人」是区分自动 / 人工的依据：显示 SYSTEM 代表系统自动审核通过；显示具体管理员账号则代表该笔为人工审核（通过或拒绝）。

弹窗字段规格

字段类型说明脱敏规则
三方订单号文本 + 复制供应商回传的订单号，供对账追溯仅显示后五码，前段以 * 遮蔽
兑换码文本（不可复制）可兑换的 PIN / 券码（码型商品）仅显示后四码，前段以 * 遮蔽；移除复制按钮
兑换链接链接（不可复制）导向供应商兑换页（链接型商品）仅显示后四码，前段以 * 遮蔽；移除复制按钮

功能逻辑

  • 兑换码 / 兑换链接依第三方 API 返回的数据显示，二者可能其一、也可能并存（取决于该商品的 API 数据）。
  • 「查看三方信息」仅 Share Treat 类型显示；其他类型维持原入口。
  • 脱敏显示：三方信息弹窗中的敏感字段均需模糊处理——三方订单号仅显示后五码，兑换码 / 兑换链接仅显示后四码；兑换码与兑换链接同时移除复制按钮，防止明文外泄。
  • 审核拒绝退积分：手动审核拒绝时，系统全额退还会员本次扣减的积分；订单状态记为「审核未通过」，操作人记为执行拒绝的管理员账号，前台同步显示「已拒绝」。
