## 0. 全局导航规则

适用于所有步骤的操作按钮行为

### 操作按钮

| 按钮 | 出现位置 | 行为 |
|---|---|---|
| 下一步 › | 步骤 1–5 | 点击后进行必填字段验证；**验证通过才可跳转至下一步骤**；有未填项则阻止跳转，高亮标示缺失字段，提示补齐后再继续 |
| ‹ 上一步 | 步骤 2–6 | 返回前一步骤，**保留已填写数据** |
| 退出编辑 ↩ | 全步骤 | 点击后弹出确认弹窗，提示文案：「确定要退出编辑吗？」<br>· 点击「**确定**」→ 返回活动列表页，已填数据不保存<br>· 点击「**取消**」→ 关闭弹窗，继续编辑 |
| ✓ 确认送出 | 步骤 6 | 执行必填验证，通过后送出并跳转回活动列表 |

> **下一步校验逻辑：** 每个步骤点击「下一步」时，仅校验当前步骤的必填字段；有未填项则阻止跳转并高亮提示。所有步骤逐一通过后，进入步骤 6 时必填项已全部完成，**步骤 6 不再重复显示必填提醒**。

### 步骤点击导航

| 操作 | 行为 |
|---|---|
| 点击步骤编号 / 步骤名称 | 只能点击**已编辑完成且通过验证**的步骤；尚未到达或未通过验证的步骤不可点击跳转 |

## 1. 活动模板设置

配置活动基本信息与时间范围

### 活动模板

| 字段名称 | 控件类型 | 必填 | 选项 / 规则 | 备注 |
|---|---|:---:|---|---|
| 活动类型 | 下拉选单 | 是 | 预约活动 / 周卡活动；无默认选项，需手动选择 | 选择「预约活动」将切换为预约活动模板样式 |
| 活动名称 | 文本输入 | 是 | 最多 30 字 | 仅供后台辨识，不显示给前台用户 |
| 活动币种 | 下拉选单 | 是 | PHP（当前仅支持 PHP） | — |
| 活动状态 | 单选 | 否 | 关闭（预选）/ 开启 | 新建时默认关闭，避免误上线；操作员手动关闭时，视同强制终止整个活动，已购买会员亦无法继续领取剩余奖励周期 |

### 时间设定

| 字段名称 | 控件类型 | 必填 | 格式 | 备注 |
|---|---|:---:|---|---|
| 进行时间 | 日期时间范围选择器 | 是 | yyyy-mm-dd hh:mm ～ yyyy-mm-dd hh:mm | 活动总持续时间；此时间区间内，已购买会员仍可完成领取奖励周期 |

## 4. 活动条件

配置各卡型奖励规则与通用活动参数

> ⚠ 修改奖励配置后将**立即生效**，已购买但尚未领取的奖励将按最新规则执行，请谨慎操作。

### 卡型配置

系统内建两种卡型，每种卡型包含 7 天签到奖励配置。

#### Classic Card（标准周卡）

| 字段名称 | 控件类型 | 必填 | 规则 | 备注 |
|---|---|:---:|---|---|
| 卡型名称 | 只读文本 | — | 固定显示 `Classic Card` | 不可修改 |
| 周卡费用（PHP） | 数字输入 | 是 | 正整数 | 默认值 `0` |
| 签到奖励（第 1–7 天） | 下拉 + 数字输入 | 是 | 类型见「奖励类型」表；数量为正整数 | 无门槛，签到即可领 |
| 额外奖励（第 1–7 天） | 下拉 + 数字输入 | 否 | 类型见「奖励类型」表；数量留空 = 该天不开放额外奖励 | — |
| 额外奖励条件（第 1–7 天） | 条件选择 + 数字输入 | 否 | 充值 ≥ N 或 投注 ≥ N；留空 = 无条件可领 | 仅在额外奖励有数量时生效 |

#### Premium Card（豪华周卡）

| 字段名称 | 控件类型 | 必填 | 规则 | 备注 |
|---|---|:---:|---|---|
| 卡型名称 | 只读文本 | — | 固定显示 `Premium Card` | 不可修改 |
| 周卡费用（PHP） | 数字输入 | 是 | 正整数 | 默认值 `0` |
| 签到奖励（第 1–7 天） | 下拉 + 数字输入 | 是 | 类型见「奖励类型」表 | 无门槛，签到即可领 |
| 额外奖励（第 1–7 天） | 下拉 + 数字输入 | 否 | 数量留空 = 该天不开放额外奖励 | — |
| 额外奖励条件（第 1–7 天） | 条件选择 + 数字输入 | 否 | 充值 ≥ N 或 投注 ≥ N；留空 = 无条件可领 | — |

**奖励类型选项（签到奖励 & 额外奖励共用）：**
彩金（Cash）／积分（Point）／免费旋转（Free Spin）／优惠券（Coupon）／免费投注（Free Bet）／道具卡（Prop Card）

**额外奖励条件选项：** 充值（Deposit）≥ N　／　投注（Turnover）≥ N

### 通用设置

| 字段名称 | 控件类型 | 必填 | 选项 / 规则 | 备注 |
|---|---|:---:|---|---|
| 存款强制弹窗 | 开关（Toggle） | 否 | 关闭（预设）/ 开启 | 开启后，玩家存款成功弹出周卡购买提示；弹窗含「今日不再显示」；点击关闭时二次确认 |
| 彩金所需流水倍数 | 数字输入 | 否 | 正整数 | 仅适用于彩金（Cash）奖励的流水稽核倍数 |
| 审核方式 | 单选 | 否 | 人工审核 / 自动审核（预选） | — |
| 领取方式 | 单选 | 否 | 自动领取（预选）/ 手动领取 | 额外奖励无论此设定，均须满足条件后由玩家手动领取 |
| 已验证手机号 | 单选 | 否 | 是 / 否（预选） | 限制仅绑定手机号的会员可参与 |
| 限制使用设备 | 单选 | 否 | 无限制（预选）/ 仅 APP | — |
| 适用游戏范围 🔴 5/25 调整 | 弹窗多选 | 否 | 点击按钮开启游戏选择弹窗；弹窗中支持依游戏类别全选 / 单选；按钮上显示已选数量，例：「已选择 35 个游戏」；**预设全选** | 指定本活动奖励条件中有效投注（Turnover）的游戏范围；玩家须在所选游戏内完成投注，才计入奖励条件达成；未选择时不限制游戏范围 |

## 5. 风控条件

设置多账号检测规则，防止刷活动行为

### 风险检测设置（此区逻辑为 OR）

| 字段名称 | 控件类型 | 必填 | 说明 |
|---|---|:---:|---|
| 同 IP 检测 | 开关（Toggle） | 否 | 检测相同 IP 地址的多个账号 |
| 同手机号检测 | 开关（Toggle） | 否 | 检测绑定相同手机号的账号 |
| 同姓名检测 | 开关（Toggle） | 否 | 检测实名认证姓名相同的账号 |
| 同 email 检测 | 开关（Toggle） | 否 | 检测绑定相同电子邮件的账号 |

### 自动拒绝（此区逻辑为 OR）

> 上方任一检测开关开启后，自动拒绝区块才可编辑；否则呈灰色不可操作。

| 字段名称 | 控件类型 | 必填 | 规则 | 备注 |
|---|---|:---:|---|---|
| 自动拒绝开关 | 开关（Toggle） | 否 | 任一检测开启时自动打开；**不可手动关闭** | 须将所有检测关闭，自动拒绝开关才会随之关闭 |
| 触发条件（笔数） | 数字输入 | 否 | 正整数；当违规笔数超过 N 笔时自动拒绝派奖 | 自动拒绝开启后可输入；关闭后清空并禁用 |

## 6. 提交预览

### 页面结构

页面由上至下依序为：步骤汇总表（步骤 1–5）。进入本步骤时，必填项已于前置步骤逐一校验完毕，本页不显示必填提醒。

| 区块 | 说明 |
|---|---|
| 步骤汇总表 | 共 5 行，对应步骤 1–5，每行左侧显示步骤编号与名称，右侧以二栏网格列出该步骤的关键字段与值 |

| 状态 | 显示方式 |

| 未填写（必填） | 显示「–」，字色红色，提示需补填 |

## 7. 关联后台页面影响
