操作按钮
按钮出现位置行为
下一步 ›步骤 1–5点击后进行必填字段验证；验证通过才可跳转至下一步骤；有未填项则阻止跳转，高亮标示缺失字段，提示补齐后再继续
‹ 上一步步骤 2–6返回前一步骤，保留已填写数据
退出编辑 ↩全步骤 点击后弹出确认弹窗，提示文案：「确定要退出编辑吗？」
· 点击「确定」→ 返回活动列表页，已填数据不保存
· 点击「取消」→ 关闭弹窗，继续编辑
✓ 确认送出步骤 6执行必填验证，通过后送出并跳转回活动列表
下一步校验逻辑：每个步骤点击「下一步」时，仅校验当前步骤的必填字段；有未填项则阻止跳转并高亮提示。所有步骤逐一通过后，进入步骤 6 时必填项已全部完成，步骤 6 不再重复显示必填提醒。

卡型配置

系统内建两种卡型，每种卡型包含 7 天签到奖励配置。

Elite Pass　标准周卡
字段名称控件类型必填规则备注
卡型名称只读文本—固定显示 Elite Pass不可修改
周卡费用（PHP）数字输入是正整数默认值 0
签到奖励（第 1–7 天）下拉 + 数字输入是类型见「奖励类型」表；数量为正整数无门槛，签到即可领
额外奖励（第 1–7 天）下拉 + 数字输入否类型见「奖励类型」表；数量留空 = 该天不开放额外奖励—
额外奖励条件（第 1–7 天）条件选择 + 数字输入否充值 ≥ N 或 投注 ≥ N；留空 = 无条件可领仅在额外奖励有数量时生效
Prime Pass　豪华周卡
字段名称控件类型必填规则备注
卡型名称只读文本—固定显示 Prime Pass不可修改
周卡费用（PHP）数字输入是正整数默认值 0
签到奖励（第 1–7 天）下拉 + 数字输入是类型见「奖励类型」表无门槛，签到即可领
额外奖励（第 1–7 天）下拉 + 数字输入否数量留空 = 该天不开放额外奖励—
额外奖励条件（第 1–7 天）条件选择 + 数字输入否充值 ≥ N 或 投注 ≥ N；留空 = 无条件可领—
奖励类型选项（签到奖励 & 额外奖励共用）：
彩金（Cash）／积分（Point）／免费旋转（Free Spin）／优惠券（Coupon）／免费投注（Free Bet）／道具卡（Prop Card）
额外奖励条件选项：充值（Deposit）≥ N　／　投注（Turnover）≥ N
通用设置
字段名称控件类型必填选项 / 规则备注
存款强制弹窗开关（Toggle）否关闭（预设）/ 开启开启后，玩家存款成功弹出周卡购买提示；弹窗含「今日不再显示」；点击关闭时二次确认
彩金所需流水倍数数字输入否正整数仅适用于彩金（Cash）奖励的流水稽核倍数
审核方式单选否人工审核 / 自动审核（预选）—
领取方式单选否自动领取（预选）/ 手动领取额外奖励无论此设定，均须满足条件后由玩家手动领取
已验证手机号单选否是 / 否（预选）限制仅绑定手机号的会员可参与
限制使用设备只读文本否无限制（固定，不可更改）—
适用游戏范围 🔴 5/25 调整 弹窗多选 否 点击按钮开启游戏选择弹窗；弹窗中支持依游戏类别全选 / 单选；按钮上显示已选数量，例：「已选择 35 个游戏」；预设全选（含体育）
※ 弹窗内需新增「体育」勾选项		 指定本活动奖励条件中有效投注（Turnover）的游戏范围；玩家须在所选游戏内完成投注，才计入奖励条件达成；未选择时不限制游戏范围
※ 体育类别不细分项目，选取即整页全部计入
5
风控条件
设置多账号检测规则，防止刷活动行为
风险检测设置 （此区逻辑为 OR）
字段名称控件类型必填说明
同 IP 检测开关（Toggle）否检测相同 IP 地址的多个账号
同手机号检测开关（Toggle）否检测绑定相同手机号的账号
同姓名检测开关（Toggle）否检测实名认证姓名相同的账号
同 email 检测开关（Toggle）否检测绑定相同电子邮件的账号
自动拒绝 （此区逻辑为 OR）
上方任一检测开关开启后，自动拒绝区块才可编辑；否则呈灰色不可操作。
字段名称控件类型必填规则备注
自动拒绝开关开关（Toggle）否任一检测开启时自动打开；不可手动关闭须将所有检测关闭，自动拒绝开关才会随之关闭
触发条件（笔数）数字输入否正整数；当违规笔数超过 N 笔时自动拒绝派奖自动拒绝开启后可输入；关闭后清空并禁用
6
提交预览
汇总展示所有步骤的配置内容，确认后送出
页面结构

页面由上至下依序为：步骤汇总表（步骤 1–5）。进入本步骤时，必填项已于前置步骤逐一校验完毕，本页不显示必填提醒。

区块说明
步骤汇总表共 5 行，对应步骤 1–5，每行左侧显示步骤编号与名称，右侧以二栏网格列出该步骤的关键字段与值
字段显示规则
状态显示方式
必填字段字段名称前标注 *
已填写正常黑色文字显示值
未填写（非必填）显示「–」，字色灰色
未填写（必填）显示「–」，字色红色，提示需补填
各步骤显示字段明细
步骤 1　活动模板设置
字段必填预览值来源
活动类型否显示选择的活动类型名称
活动状态否显示「开启」或「关闭」
活动币种否显示选择的币种（如 PHP）
活动名称是显示输入的名称；未填则红色「–」
进行时间是显示起讫时间区间；未填则红色「–」
步骤 2　活动图片
字段必填预览值来源
首页浮窗否显示「开启」或「关闭」
排序否显示输入的数字，默认「0」
备注否显示输入内容；未填则灰色「–」
步骤 3　参与对象与名单
字段必填预览值来源
充值层级否显示已选层数，例：「全选（7 层）」
指定代理线否显示代理线名称；未指定则显示「不指定」
新会员限制否显示「是」或「否」
会员等级否显示已选范围，例：「全选（VIP0–VIP10）」
黑名单否显示已设定内容；未填则灰色「–」
步骤 4　活动条件
字段必填预览值来源
存款强制弹窗否显示「开启」或「关闭」
彩金流水倍数否显示输入数字；未填则灰色「–」
审核方式否显示「人工审核」或「自动审核」
领取方式否显示「自动领取」或「手动领取」
已验证手机号否显示「是」或「否」
限制使用设备否固定显示「无限制」
适用游戏范围否显示已选游戏数量，例：「26 个」；未选则灰色「–」
卡型奖励摘要—每张卡型独立一行，显示：卡名 · 费用（PHP）· 有额外奖励的天数，例：「Elite Pass · 500 PHP · 3 天有额外奖励」
步骤 5　风控条件
字段必填预览值来源
同 IP 检测否显示「开启」或「关闭」
同手机号检测否显示「开启」或「关闭」
同姓名检测否显示「开启」或「关闭」
同 email 检测否显示「开启」或「关闭」
自动拒绝否显示「开启 · 触发 N 笔以上」或「关闭」；未设定则灰色「–」
确认送出行为
情境系统行为
提交成功提交活动配置至后端，成功后跳转至活动列表页
提交失败（后端异常）显示错误提示，页面停留在步骤 6，不清空已填内容
